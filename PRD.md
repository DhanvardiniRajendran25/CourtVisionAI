# PRD: CourtVision AI — Scout Agent Backend

## Overview

Backend service for CourtVision AI's Scout Agent. A FastAPI server that lets basketball coaches ask scouting questions in plain English and get tactical responses grounded in real current-season stats via Gemini Flash + Google Search Grounding.

This service is also the **shared backend foundation** — it sets up the FastAPI project, CORS, folder structure, and patterns that Video Analyzer and Simulator endpoints plug into.

## What This Service Does

1. **Accepts natural language basketball scouting questions** from coaches
2. **Calls Gemini 2.5 Flash with Google Search Grounding** to get answers backed by real current-season data
3. **Maintains multi-turn conversation memory** per session (in-memory dict)
4. **Returns structured responses** with: answer text, confidence score, source citations, search queries used, and suggested follow-up questions
5. **Compiles scouting intel** and makes it available for the Simulator agent via a "send to sim" endpoint
6. **Provides stub endpoints** for Video Analyzer and Simulator so frontend can develop against the full API surface immediately

## API Contract

### `GET /health`

Health check for deployment verification.

**Response:**
```json
{ "status": "ok" }
```

### `POST /api/scout/ask`

Main scout endpoint. Coach asks a question, gets a grounded tactical answer.

**Request Body:**
```json
{
  "question": "What's Duke's biggest weakness this season?",
  "session_id": "scout_abc123",
  "coach_context": {
    "team": "Auburn",
    "injured_players": ["Johni Broome"],
    "style": "uptempo"
  }
}
```

- `question` (required): The coach's scouting question in plain English
- `session_id` (optional, default "default"): Groups messages into a conversation for multi-turn follow-ups
- `coach_context` (optional): Coach's own team info for personalized advice. AI won't suggest strategies relying on injured players.

**Response Body:**
```json
{
  "answer": "Duke's biggest vulnerability this season is perimeter defense. They rank 187th nationally in 3PT defense, allowing opponents to shoot 35.2% from deep...",
  "confidence": 0.85,
  "sources": [
    {
      "title": "ESPN Duke Blue Devils Stats",
      "uri": "https://www.espn.com/..."
    },
    {
      "title": "KenPom Rankings",
      "uri": "https://kenpom.com/..."
    }
  ],
  "search_queries": ["Duke basketball weaknesses 2025-2026 season stats"],
  "suggested_followups": [
    "How do they perform in the second half?",
    "Who is their weakest perimeter defender?",
    "What's their record against top-25 teams?"
  ],
  "session_id": "scout_abc123"
}
```

- `confidence`: Float 0-1. Derived from how well the response is grounded (number of supporting sources, specificity of stats found). Hardcode a heuristic: if grounding_chunks >= 3 → 0.9, 2 → 0.8, 1 → 0.7, 0 → 0.5.
- `sources`: Extracted from Gemini's `groundingMetadata.groundingChunks`. Each has a `title` and `uri`.
- `search_queries`: From `groundingMetadata.webSearchQueries`. Show in UI for transparency.
- `suggested_followups`: Generated by the model as part of the response (prompted to include them).

### `POST /api/scout/send-to-sim`

Compiles all scouting intel from a session into a summary the Simulator agent can use.

**Request Body:**
```json
{
  "session_id": "scout_abc123",
  "team": "Duke"
}
```

**Response Body:**
```json
{
  "intel_summary": "DUKE SCOUTING INTEL:\n- Weak perimeter defense (35.2% opponent 3PT)\n- Turnover-prone under full court press (13.2 TO/game)\n- Strong in transition (22.4 fast break pts/game)\n- Key player: [name] averages 18.5 PPG but struggles from FT line (67%)\n...",
  "status": "ready_for_simulator"
}
```

## System Prompt

The Scout Agent uses this system prompt with every Gemini call:

```
You are CourtVision AI Scout, an elite basketball scouting assistant.

You help coaches prepare for upcoming opponents by providing tactical intelligence
grounded in REAL, CURRENT-SEASON statistics.

RESPONSE FORMAT — You MUST structure every response as follows:

1. DIRECT ANSWER: Lead with the tactical answer to the coach's question. Be specific.
   Use real numbers. Name players. Cite stats.

2. KEY STATS: Pull 3-5 most relevant statistics that support your answer.

3. TACTICAL RECOMMENDATION: Based on the data, give 1-2 actionable coaching suggestions.
   If coach_context is provided, tailor recommendations to their team and style.
   NEVER suggest strategies relying on players listed as injured.

4. CONFIDENCE NOTE: Rate your confidence (High/Medium/Low) based on data availability.
   High = multiple current sources with specific stats.
   Medium = some data but incomplete picture.
   Low = limited data, more speculative.

5. SUGGESTED FOLLOW-UPS: End with exactly 3 follow-up questions the coach might want
   to ask next. Format as a JSON array in a line starting with "FOLLOWUPS:"
   Example: FOLLOWUPS: ["How do they perform in close games?", "Who is their best 3PT shooter?", "What defense do they run most?"]

RULES:
- Use 2025-2026 season data when available
- Always cite specific numbers (don't say "they struggle" — say "they rank 234th in turnover margin at -2.3/game")
- Frame everything from a coaching perspective
- Be concise. Coaches are busy.
- If asked about a matchup, analyze both teams
```

## Gemini SDK Usage

**Client initialization (use google-genai, NOT the deprecated google-generativeai):**

```python
from google import genai
from google.genai import types

client = genai.Client(api_key=GEMINI_API_KEY)
```

**Making a grounded search call:**

```python
google_search_tool = types.Tool(google_search=types.GoogleSearch())

response = client.models.generate_content(
    model="gemini-2.5-flash",
    contents=contents,  # list of types.Content for multi-turn
    config=types.GenerateContentConfig(
        system_instruction=SCOUT_SYSTEM_PROMPT,
        tools=[google_search_tool],
        temperature=0.2,
        max_output_tokens=3000,
    )
)
```

**Extracting grounding metadata from response:**

```python
# Answer text
answer = response.text

# Sources
sources = []
grounding_meta = response.candidates[0].grounding_metadata
if grounding_meta and grounding_meta.grounding_chunks:
    for chunk in grounding_meta.grounding_chunks:
        if chunk.web:
            sources.append({
                "title": chunk.web.title or "",
                "uri": chunk.web.uri or ""
            })

# Search queries used
search_queries = []
if grounding_meta and grounding_meta.web_search_queries:
    search_queries = list(grounding_meta.web_search_queries)
```

## Session Management

In-memory dict keyed by `session_id`. Each session stores conversation history as a list of role/content pairs for multi-turn.

```python
sessions: dict[str, list[dict]] = {}

# Structure per session:
# sessions["scout_abc123"] = [
#     {"role": "user", "text": "What's Duke's weakness?"},
#     {"role": "model", "text": "Duke's biggest vulnerability..."},
#     {"role": "user", "text": "How about their free throw shooting?"},
#     {"role": "model", "text": "Duke shoots 71.2% from the line..."},
# ]
```

When building the `contents` for a Gemini call, convert session history into `types.Content` objects:

```python
contents = []
for msg in session_history:
    contents.append(
        types.Content(
            role=msg["role"],
            parts=[types.Part(text=msg["text"])]
        )
    )
# Append new question
contents.append(
    types.Content(role="user", parts=[types.Part(text=question)])
)
```

## Confidence Score Heuristic

Don't ask the model for a confidence number — derive it from grounding quality:

```python
def calculate_confidence(grounding_metadata) -> float:
    if not grounding_metadata or not grounding_metadata.grounding_chunks:
        return 0.5
    num_sources = len(grounding_metadata.grounding_chunks)
    if num_sources >= 3:
        return 0.9
    elif num_sources == 2:
        return 0.8
    elif num_sources == 1:
        return 0.7
    return 0.5
```

## Follow-up Extraction

The system prompt tells the model to end with `FOLLOWUPS: [...]`. Parse it:

```python
import json, re

def extract_followups(answer_text: str) -> tuple[str, list[str]]:
    """Extract follow-up suggestions and return clean answer + followups."""
    followups = []
    clean_answer = answer_text

    match = re.search(r'FOLLOWUPS:\s*(\[.*?\])', answer_text, re.DOTALL)
    if match:
        try:
            followups = json.loads(match.group(1))
            clean_answer = answer_text[:match.start()].strip()
        except json.JSONDecodeError:
            pass

    if not followups:
        followups = [
            "Tell me more about their key players",
            "How do they perform under pressure?",
            "What's their record in away games?"
        ]

    return clean_answer, followups
```

## Send-to-Sim Endpoint

Compiles all model responses from a scout session into a condensed intel brief:

1. Grab all model responses from the session history
2. Send them to Gemini Flash with a summarization prompt:
   "Compile this scouting conversation into a concise tactical intel brief about {team}. Format as bullet points. Include: strengths, weaknesses, key players, tendencies, and recommended strategies against them."
3. Store the result in a shared `intel_store` dict that the Simulator can read
4. Return the summary

## Error Handling

- Gemini API timeout → return `{"answer": "Scouting request timed out. Please try again.", "confidence": 0, ...}`
- Invalid/empty question → 400 with message
- Gemini rate limit → 429 with retry-after
- Any unhandled exception → 500 with generic message, log the actual error

## What Frontend Needs to Know

1. **Base URL:** `http://localhost:8000` (local) or Cloud Run URL (deployed)
2. **All endpoints accept JSON** with `Content-Type: application/json`
3. **CORS is open** (`allow_origins=["*"]`) — no issues calling from any frontend origin
4. **Swagger docs** at `/docs` — frontend team can explore and test all endpoints
5. **Session IDs** are client-generated — frontend should create a unique ID per scout conversation (e.g. `scout_${Date.now()}`)
6. **`coach_context` is optional** — app works without it but gives better answers with it
